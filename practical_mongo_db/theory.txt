# 1. Introduction to Database

Before understanding MongoDB, first understand what a database is.

A **database** is a place where data is stored in an organized manner.

Example:

* Student records
* Employee data
* Product details
* Banking information
* Social media posts

---

# 2. Types of Databases

Databases are mainly divided into:

## A. Relational Database (SQL)

Examples:

* MySQL
* Oracle Database
* PostgreSQL

Uses:

* Tables
* Rows
* Columns
* Fixed schema
* SQL language

Example table:

| ID | Name | Age |
| -- | ---- | --- |
| 1  | John | 25  |

---

## B. NoSQL Database

Examples:

* MongoDB
* Cassandra
* Redis

Uses:

* Flexible structure
* High scalability
* Fast handling of huge data
* JSON-like documents

MongoDB belongs to this category.

---

# 3. What is MongoDB?

MongoDB is a:

* NoSQL database
* Document-oriented database
* Open-source database
* High-performance database

MongoDB stores data in the form of **documents** instead of tables.

---

# 4. Why MongoDB is Used

MongoDB is used because modern applications generate:

* Huge data
* Unstructured data
* Dynamic data
* Real-time data

Traditional SQL databases become difficult for:

* Frequent schema changes
* Big data scaling
* High-speed applications

MongoDB solves these problems.

# 8. Important MongoDB Terms

---

## A. Database

A database contains collections.

Example:

```text
StudentDB
EmployeeDB
HospitalDB
```

Command:

```javascript
use StudentDB
```

---

## B. Collection

A collection is similar to a table in SQL.

It stores documents.

Example:

```text
students
employees
products
```

---

## C. Document

A document is a single record.

Stored in JSON/BSON format.

Example:

```json
{
  "name": "Krishna",
  "age": 22,
  "city": "Pune"
}
```

---

## D. Field

Field means key-value pair.

Example:

```json
"name": "Krishna"
```

Here:

* name = field
* Krishna = value

# 19. CREATE Operation

Used to insert data.

---

## insertOne()

Inserts one document.

Example:

```javascript
db.students.insertOne({
   name: "Krishna",
   age: 22,
   city: "Pune"
})
```

How it works:

1. MongoDB checks collection
2. Converts JSON to BSON
3. Generates `_id`
4. Stores document

Result:

```json
{
 "_id": ObjectId(...),
 "name": "Krishna",
 "age": 22
}
```

---

## insertMany()

Insert multiple documents.

Example:

```javascript
db.students.insertMany([
 {name:"A"},
 {name:"B"},
 {name:"C"}
])
```

---

# 20. READ Operation

Used to retrieve data.

---

## find()

Returns multiple documents.

Example:

```javascript
db.students.find()
```

---

## findOne()

Returns single document.

Example:

```javascript
db.students.findOne({name:"Krishna"})
```

---

## Query Operators

---

### Greater Than

```javascript
db.students.find({age: {$gt: 20}})
```

Meaning:
Find students whose age > 20

---

### Less Than

```javascript
db.students.find({age: {$lt: 30}})
```

---

### Equal

```javascript
db.students.find({name:"John"})
```

---

### AND Condition

```javascript
db.students.find({
 age: {$gt:20},
 city:"Pune"
})
```

---

### OR Condition

```javascript
db.students.find({
 $or:[
   {city:"Pune"},
   {city:"Mumbai"}
 ]
})
```

---

# 21. UPDATE Operation

Used to modify existing data.

---

## updateOne()

Example:

```javascript
db.students.updateOne(
 {name:"Krishna"},
 {$set:{age:23}}
)
```

How it works:

1. Finds matching document
2. Updates field
3. Saves modified document

---

## updateMany()

Example:

```javascript
db.students.updateMany(
 {city:"Pune"},
 {$set:{state:"Maharashtra"}}
)
```

---

## Increment Operator

```javascript
db.students.updateOne(
 {name:"John"},
 {$inc:{age:1}}
)
```

---

Without index:

* MongoDB scans all documents

With index:

* Fast searching

# 29. Sharding
